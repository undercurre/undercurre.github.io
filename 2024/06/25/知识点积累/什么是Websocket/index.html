什么是 Websocket?

Websocket 它是一种在单个 TCP 链接上形成全双工通信的协议，允许客户端和服务器之间进行双向实时通信。那么在 Websocket 之前，大部分应用它如果想实现这种实时更新，就不得不依赖于 HTTP 长轮询或者短轮询的方式不断的向服务器发送请求来获取最新的数据。

那么 Websocket 对于这一种长轮询，它有哪些优势呢？

首先第一个就是持久链接，一旦这个 Websocket 的链接成功，那么客户端和服务器之间他们的这个链接将持续存在，直到任意一方主动关闭链接它。这就不同于 HTTP 请求，每次请求都需要重新建立链接，然后接收响应后关闭链接。

第二个就是双向通信, Websocket，它是支持双向通信，也就是说允许服务器主动向客户端推送消息，那也支持客户端向服务器发送请求。而 HTTP 长轮询，尽管服务器可以在接收到客户端请求后保持链接并推送数据，但仍然是服务器响应客户端请求的模式，本质上还是服务器被动响应。

第三个低延迟 Websocket，它是不需要反复建立链接和关闭链接，那这样的话就减少了 HTTP 头部的一个开销，因此在频繁交互的情况下，它的延迟更低，实时性更强。

第四个就是减少资源消耗，那么它相对于 HTTP 长轮询， Websocket 在维持链接状态时只需占用一个链接，而长轮询可能需要不断的发起和关闭链接，尤其在并发数量比较大的时候，那服务器资源利用率就会更高了。

第五个就是消息帧。Websocket 的协议自身定义了一套消息帧格式，能够有效的传输数据，包括二进制数据，而 HTTP 它是基于文本报文，而二进制数据的支持相对比较复杂。总的来说， Websocket 它提供了一种更有效、实时性更好的通信方法，尤其适用于实时数据传输、聊天应用、实时图表更新游戏以及其他这个低延迟双向通信的一些应用场景。相对于长轮询 websocket 无需反复建立链接，降低了延迟和带宽消耗，提高服务器和客户端的影响率。

说一下传统的长轮询和短轮询

短轮询是一种传统的轮询方式，客户端会定期向服务器发送请求以获取最新的数据。譬如客户端按固定的时间间隔（例如每秒一次）发送 HTTP 请求询问服务器是否有新的数据。
长轮询是一种改进的轮询方式，客户端发送请求后，如果服务器没有新数据，会保持连接直到有新数据或超时后再返回响应。客户端发送一个请求，服务器在有新数据可用时立即响应。如果没有数据，服务器保持连接一段时间（例如 30 秒），在此期间有数据则立即响应，没有数据则超时返回空响应。客户端在收到响应后立即发送下一个请求。
