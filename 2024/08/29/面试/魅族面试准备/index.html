魅族面试准备

魅族面试准备

1. CSR，SSR 和 SSG 是什么，有什么优缺点？

CSR 就是客户端渲染，可以前后端分离，减少服务端压力，可以做 SPA，SPA 的理念是增量渲染，对用户交互更友好。但是 seo 不好，首屏时间比较长。

SSR 是服务器渲染，有利于 SEO，有利于首屏渲染，但是会占用服务器资源，交互要跳页面，就浏览器会刷新。

SSG 就是静态站点。SEO 好且可以放 CDN，但只适用于静态数据，交互也触发浏览器刷新。通常跟 cdn 组合使用。

浏览器输入 URL 后的执行过程?

1、用户输入网址后，浏览器发起 DNS 查询请求；2、建立 TCP 连接；3、发送 HTTP 请求；4、服务器处理请求；5、返回 HTTP 响应；6、释放 TCP 连接；7、浏览器解析返回的资源和数据。(1) html 文档解析 dom 树(2) 异步获取图片和 css 资源，js 会阻塞(3) css 解析样式层叠树(4) js 解析(5)交互过程中产生重排（layout 改变）和重绘（css 属性改变）

在浏览器中，为什么 JavaScript 的执行会阻塞渲染？

渲染线程是 UI 线程，js 线程是主线程，他们虽然是两个线程，但是是互斥的，因为浏览器要避免渲染结果和 DOM 操作不一致

知道 async 和 defer 属性吗？

async 属性: 当使用 <script> 标签的 async 属性时，JavaScript 文件会被异步下载并执行，不会阻塞 HTML 的解析，但执行时仍会阻塞渲染。

defer 属性: 使用 defer 属性时，JavaScript 文件同样会被异步下载，但执行会被延迟到 HTML 解析完成之后，从而不会阻塞渲染。

没有这两个属性情况下，会阻塞 html 解析，所以尽量放在 body 后面

你知道哪些 CSS 的优化？

内联首屏关键 CSS，关键代码能够使浏览器在下载完 html 后就能立刻渲染
用 webpack 进行 css 资源压缩，让文件更小
尽可能使用高优先级的选择器，最多不超过三层索引
减少使用昂贵的属性，尽量重绘不要重排，transition 代替 width 和 height 的更改
不要使用@import，因为会等页面下载完再加载。

首屏优化要怎么做？

  1. 将 JavaScript 脚本放在<body>的最后面，避免资源阻塞页面渲染
  2. 拆包，然后后续资源懒加载或者预加载（闲事加载），gzip 压缩，资源合并像雪碧图
  3. 使用 cdn 缓存
  4. 请求合并，减少请求数量，提升请求并发。

你平时如何监测页面性能？

就是用 Chrome DevTools，从时间和空间上进行分析，时间上关注页面加载耗时、渲染耗时、网络耗时、脚本执行耗时，空间上关注 CPU 占用、内存占用、本地缓存占用，

具体有 lighthouse 可以得到性能报告，performance 可以录制和分析运行时性能，分析帧动态和线程状态，FPS 标红为低帧率长任务, 需要任务分片处理; CPU 火焰图和 Bottom-up 能看到内存性能瓶颈的具体函数，通常需要优化处理大量数据的算法。

缓存头

Expires

当客户端第一次访问一个文件资源的时候，服务端在返回资源内容的同时也返回了

Expires: Mon, 1 Aug 2016 22:43:02 GMT

也就是服务端告诉浏览器，先把这个文件缓存起来，在这个过期时间之前，该文件都不会变化了。

Cache-Control

给 expires 一个相对时间，用来避免用户系统时间和真实时钟不一致的问题

Last-Modified / If-Modified-Since

服务端有个文件可能会更新，因此希望客户端时不时过来问一下这个文件是否过期。

如果没有过期，服务端不返回数据给浏览器，只返回 304 状态码，告诉浏览器目前的缓存还没有过期，然后浏览器继续使用已有缓存。

这个就叫做条件请求。

Etag / If-None-Match

条件查询另一种判断方式，通过 tag 来打标签，判断资源是否变更。

cdn 是什么？

用节点服务器做均衡负载，让用户请求到最近的资源，具体操作就像 cloudfare 配一下 节点服务器的 ip

xss,csrf 的漏洞和防范

xss 跨站脚本攻击，csrf 跨站请求伪造，xss 多出现在 v-html 这种直接把文本跑成代码的地方，就像评论什么的被输入一段 script 标签源代码，然后访问黑客服务器，然后就能在请求拿到用户的 cookie，csrf 多出现在，给用户发一条信息，里面包含一个假网站，然后把请求封在 img 的 src 里，用户点开就自动使用浏览器上的凭证发起请求，完成转账，买东西，修改密码等高危操作。xss 防护主要靠服务器过滤提交的文本不能是源代码，cookie 使用 httponly 属性避免被 js 提取到。csrf 防护主要靠 Cookie 的 SameSite，让浏览器禁止第三方 Cookie。以及使用 crsf token 来确保请求合法。

devicePixelRatio 的处理

devicePixelRatio = 物理像素 / 设备独立像素（device-independent pixel 简称 dpis）

响应式图片

react 和 vue 的区别？diff 算法有什么不同？

React 是 Fiber 架构的没有反向指针，从右往左遍历链表会很困难。

Vue 无论 2 还是 3 都支持双端检测

共同点：Vue2 在 Diff 的时候与 React 在 Diff 的时候都采用了 map 来加快查找的效率

react 的 diff 会逐个遍历 newChildren 与 older 做对比，看能否复用， 能复用就移动，type 不同就删除和插入新 DOM 元素

vue 是对比双端后就进行移动，再对比则更新节点，Vue3 还构造了最长递增子序列，最大程度降低了 DOM 操作

scss， postcss 等 css 处理器的用途？

scss 作为预处理器用于结构化 css，postcss 作为后处理器用于提高兼容性，譬如匹配不同浏览器的前缀

怎么算第三方 cookie？

域名不同就算，二级域名及其他子域不算

cookie 的 secure 属性？

Secure 属性指定浏览器只有在加密协议 HTTPS 下，才能将这个 Cookie 发送到服务器。

video 相关？

自动播放需要静音

使用 preload-auto，但是需要带宽不然会让首屏变慢。
