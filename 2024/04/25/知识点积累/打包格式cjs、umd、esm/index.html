打包格式 cjs、umd、esm

这三种格式（ESM、UMD 和 CommonJS）是用于在不同环境下导出和使用 JavaScript 模块的标准。它们各自具有不同的特点和适用场景：

1. ESM(ES Module)

特点：ESM 是 ECMAScript 模块的标准格式，它使用 import 和 export 语法来定义模块之间的依赖关系。

用途：主要用于现代浏览器和支持 ES6 模块的打包工具中，比如 Webpack、Rollup 等。ESM 的优势在于它可以进行静态分析和优化，使得代码加载更快，并且支持树摇（tree shaking）等高级特性。

2. UMD(Universal Module Definition)

特点：UMD 是一种通用模块定义格式，它兼容了 CommonJS、AMD 和全局变量（Global Variable）等多种模块系统，可以在浏览器环境和 Node.js 环境中使用。

用途：主要用于在不同环境下都能够使用的库或组件，UMD 可以通过 require、define 或者全局变量的方式进行导入和使用。但它并不支持静态分析和优化，因此在打包工具中可能无法发挥最佳性能。

3. CJS(CJS)

特点：CommonJS 是 Node.js 默认的模块系统，它使用 require 和 module.exports 来定义模块。

用途：主要用于 Node.js 环境中，用于编写服务端代码或者构建工具。CommonJS 模块在 Node.js 环境中表现良好，但在浏览器环境中需要通过打包工具转换为其他格式才能使用。

如何选择适合的模块格式？

  1. 如果你的目标是在现代浏览器中运行，并且希望利用 ES6 模块的优势，那么使用 ESM 格式是最好的选择。
  2. 如果你的目标是编写通用的库或组件，可以在浏览器和 Node.js 环境中使用，并且不想引入额外的构建步骤，那么使用 UMD 格式是比较合适的。
  3. 如果你的目标是在Node.js 环境中编写服务端代码或者构建工具，并且不需要在浏览器中运行，那么使用 CommonJS 格式是最方便的。
