自动集成

自动集成

需求澄清

实现自动化部署工具，只需点击一个按钮，就可以执行服务器部署脚本，完成功能更新迭代。

技术方案

客户端

  1. 静态 html + css，非前端工程化，库都以 cdn 形式引入，通过库以 UMD 打包方式暴露的全局变量使用
  2. vue3，MVVM 框架，不用操作 dom
  3. element-plus，基础表单样式统一、美化
  4. axios，请求接口
  5. socket.io，接收实时部署 log

服务端

  1. 普通接口，可能需要等完全部署好后，才能拿到结果
  2. 基于 Node.js 技术栈，无数据库
  3. commander，用于生成的命令 zuodeploy 运行时帮助文档、提示，zuodeploy start 执行入口
  4. prompts，参照 vue-create，引导用户输入端口、密码
  5. koa，http 服务端, 提供接口、静态服务运行容器（类似 nginx、tomcat 等）
  6. koa-bodyparser，用于解析 post 请求参数（login 鉴权接口需要）
  7. koa-router，用于不同接口（路径，比如 /login, /deploy 等）执行不同的方法
  8. koa-session，用于接口鉴权，防止他人获取到部署接口后疯狂请求部署
  9. koa-static，静态服务器，类似 nginx 启动静态服务
  10. socket.io，socket 服务端，当 git pull, npm run build 部署时间较长时，实时发送 log 到前端
  11. log4js，带时间戳的 log 输出
  12. pm2，直接执行，当 terminal 结束服务会被关掉，用 pm2 以后台方式静默执行
