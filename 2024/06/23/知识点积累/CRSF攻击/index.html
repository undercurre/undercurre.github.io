CRSF攻击

CRSF 攻击是什么？

它是跨站请求伪造，也是一种网络安全威胁，它利用用户在特定网站上的已登录状态，诱导用户在不知情的情况下执行攻击者预定的一些操作。简单来说，攻击者通常通过这个受害者浏览器发送一个恶意请求到受害者已经登录的网站。那么由于浏览器会自动携带这个网站的 cookie 这些认证信息网站就误认为请求是用户自己发起的，从而执行恶意的操作，比如说转账、修改密码、发布内容等等。这个就是 CSRF 攻击。

那我们该如何防范？

比如说使用 token 验证，对吧？验证，那么在表单或者 ajax 请求当中加入一个随机生成的一次性使用的 token 服务器，在接收到请求时，除了验证 cookie 外，还会验证这个 token 是否正确，并且没有被使用过。 token 通常存储在这个 HTTPOnly 的 cookie 中，或者作为隐藏字段嵌入到表单里面，避免通过 JS 被读取到。

那么第二种方法我们也可以验证这个 。这些我们可以检查请求头当中的 Referer 字段，确认请求是来自预期的这个站点，也可以实施用户交互确认。对于一些敏感的操作，要求用户进行二次确认，比如输入密码、使用验证码等等。虽然这可能影响用户体验，但在某些高风险场景下是必要的。
