BigInt类型

BigInt

什么时候出现？

BigInt 类型是在 ECMAScript 2020 标准中引入的，该标准于 2020 年 6 月发布。BigInt 类型用于表示任意精度的整数，可以处理超过 Number 类型范围的整数值。

背景？解决了什么问题

在 JavaScript 中，Number 类型使用 IEEE 754 标准来表示数字，它使用 64 位浮点数格式来存储数字，因此有一个最大安全整数值（Number.MAX_SAFE_INTEGER），它是 2 的 53 次方减
1，即 9007199254740991。如果超出这个范围，就会丢失精度或者变成 Infinity。

64 位浮点数格式

64 位浮点数包含 3 个部分：

符号位（1 位）：用于表示数字的正负性。0 表示正数，1 表示负数。
指数位（11 位）：用于表示数字的大小范围。它是一个偏移值，表示小数点在尾数中向左或向右移动的位数。
尾数位（52 位）：用于表示数字的精度。它是一个 52 位的二进制数，表示小数部分的精度。

其实就是你把要表达的数字先变成科学计数法再存在相应的二进制结构里

举个例子

114 = 1.14 * 10^2

符号位 就是正号

指数位 就是 2

尾数位 就是 14

BigInt 类型可以解决这个问题，它使用任意精度算法来存储整数，可以处理非常大的整数值。BigInt 类型的字面量表示方式是在数字后面加上一个 “n”，例如：const bigIntValue = 9007199254740992n。

需要注意的是，BigInt 类型与 Number 类型是不同的，它们之间不能直接进行数学运算，需要使用 BigInt 类型的方法和运算符。另外，BigInt 类型也不能与其他类型进行比较，需要使用 BigInt 类型的比较运算符（=== 和 !==）。
