实现页面的重新加载

如何使用 JS 实现页面的重新加载

使用 location 对象的 reload 方法来实现。调用这个方法去刷新当前页面时，有两种方向，一种是立即加载，从浏览器缓存中加载，另一种是强制从服务器重新加载，忽略本地缓存。

如果说你要强制从服务器重新加载，忽略本地缓存，是吧？那就是强制从服务器加载，那你就要调用 reload 的时候给它传入一个 true。那么像第一种这种不带参数的页面，它是重新加载，浏览器可能会使用本地缓存当中的资源来加快这个加载速度，那么这也就意味着如果页面资源没有变化，用户可能不会看到实际的服务器更新，那么你加上这个 true 以后，就是让浏览器绕过本地缓存，强制从服务器重新请求所有资源。

那么这对于确保用户获取服务器上的最新内容是比较管用的，尤其是在内容频繁更新的一些场景下，但是这里面是需要我们注意一些点的，你比如说在某些情况下频繁或者不必要的页面重新加载，可能会影响用户的体验，所以说一定要谨慎的使用这个功能。另外就是当页面包含表单数据或者用户输入的时候，重新加载可能会导致数据丢失，除非采取措施先保存了这些数据。

最后就是一定要考虑到性能和用户体验，尽量通过 ajax 或者其他前端技术来实现部分页面内容的动态更新，而不是整体页面的一个重新加载。通过这个方法我们就可以灵活的在 JS 当中控制页面的重新加载行为，满足不同的这个场景需求。OK，这个就是实现页面重新加载的方法了。
