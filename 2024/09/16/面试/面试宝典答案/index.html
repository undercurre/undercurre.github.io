闭包里面的变量为什么不会被垃圾回收 ?

只有当闭包函数的引用计数不为 0 时，说明闭包函数随时有可能被调用，他被调用后，就会引用他在定义时所处的环境的变量。闭包中的变量就得一直需要在内存中，则就不会被垃圾回收掉。譬如一个在全局变量环境下的闭包，全局变量在浏览器关闭之后会回收，所以如果不设置为 null，内存就会一直分配给这个闭包中的变量。

说一下对闭包的理解 ?

其实就是利用作用域嵌套让函数可以记住并访问其词法作用域,譬如一个函数 A，return 其内部的函数 B，被 return 出去的 B 函数能够在外部访问 A 函数内部的变量，这时候就形成了一个 B 函数的变量闭包，A 函数执行结束后这个变量背包也不会被销毁，并且这个变量闭包在 A 函数外部只能通过 B 函数访问。

浅拷贝和深拷贝的区别 ？

浅拷贝是创建一个新对象，这个对象有着原始对象属性值的一份精确拷贝。如果属性是基本类型，拷贝的就是基本类型的值，如果属性是引用类型，拷贝的就是内存地址 ，所以如果其中一个对象改变了这个地址，就会影响到另一个对象。浅拷贝只复制对象或数组的第一层，对于嵌套的对象或数组，浅拷贝只复制它们的引用，而不复制实际的对象或数组。

深拷贝是将一个对象从内存中完整的拷贝一份出来,从堆内存中开辟一个新的区域存放新对象,且修改新对象不会影响原对象。

for in 和 for of 的区别和原理？

for…in 循环用于遍历对象自身的可枚举属性以及通过原型链继承的可枚举属性。

它适用于对象，但不适用于遍历数组，因为它不仅遍历数组元素，还会遍历原型链上的属性。

for…of 循环用于遍历可迭代对象（如数组、字符串、Map、Set 等）。

它不能遍历对象的属性（包括自身的和原型链上的）。

什么是可枚举属性？

Object.defineProperty 第三个参数是一个属性描述符对象，用于定义或修改对象的属性。
这个描述符对象可以包含以下几个可选的键：
value：属性的值。
writable：一个布尔值，表示属性的值是否可以被改变。
enumerable：一个布尔值，表示属性是否可以在 for…in 循环和 Object.keys 方法中被枚举。
configurable：一个布尔值，表示属性是否可以被删除，以及除 value 和 writable 之外的属性描述符是否可以被修改。
get：一个函数，作为属性的 getter，如果没有 getter 则为 undefined。当访问该属性时，这个函数会被调用，函数的返回值会被当作属性的值。
set：一个函数，作为属性的 setter，如果没有 setter 则为 undefined。当属性值被修改时，这个函数会被调用，并传入修改后的新值。

说说 js 中的栈和堆

栈是一种后进先出（LIFO, Last In First Out）的数据结构。它通常用于管理函数调用和局部变量。栈的操作是非常快速的，因为它遵循 LIFO 原则，意味着最近添加的数据项最先被移除。

特点：
有序结构：栈中的数据按顺序排列，后添加的数据项在栈顶。
自动管理内存：当函数调用时，分配内存空间，当函数执行完毕，自动释放内存。
快速访问：由于数据的添加和移除都在栈顶进行，所以操作非常快速。

堆（Heap）
堆是一种无序的内存结构，用于存储对象和动态分配的内存。与栈不同，堆中的内存是随机分配的，因此在访问速度上可能会比栈慢一些。

特点：
无序结构：堆中的数据没有特定的顺序，分配和释放内存的方式较为灵活。
手动管理内存：虽然现代语言（如 JavaScript）有垃圾回收机制，但仍需要注意内存泄漏和手动管理内存。
灵活性：堆适合用于存储大小不固定的数据结构，如对象、数组等。

网络基础

介绍下 http1.0、http1.1、http2.0 协议的区别？

http1.0 的特点是每次请求和响应完毕后都会销毁 TCP 连接，
http1.1 是 http1.0 的改进版, 能够长连接，在 HTTP/1.x 中，由于浏览器对每个域名的并发连接数有限（通常是 6 个）
http2.0 有多路复用提高吞吐量，同时支持二进制传输（2.0 之前是文本传输）。HTTP/2.0 通过引入二进制帧、多路复用和流的概念，使得同一连接上可以并行传输多个请求和响应，而不受制于请求的顺序性。

http 和 https 的区别

  1. HTTP 明文传输，数据都是未加密的，安全性较差，HTTPS（SSL+HTTP） 数据传输过程是加密的，安全性较好。
  2. 端口不一样，http 是 80，https 是 443
  3. 使用 HTTPS 协议需要到 CA（Certificate Authority，数字证书认证机构） 申请证书（在服务端）。
  4. HTTP 页面响应速度比 HTTPS 快，因为多了 ssl 握手
  5. HTTPS 其实就是建构在 SSL/TLS 之上的 HTTP 协议，所以，要比较 HTTPS 比 HTTP 要更耗费服务器资源。
  6. 总的来说，https 比 http 多了一个 rsa+aes 的混合加密过程

前端如何实现即时通讯？

  1. 短轮询。即客户端每隔一段时间就向服务器发送消息，询问有没有新的数据
  2. 长轮询，发起一次请求询问服务器，服务器可以将该请求挂起，等到有新消息时再进行响应。响应后，客户端立即又发起一次请求，重复整个流程。
  3. websocket，握手完毕后会建立持久性的连接通道，随后服务器可以在任何时候推送新消息给客户端

如何解决跨域问题？

同源策略是指只有具有相同源的页面才能够共享数据，比如 cookie，同源是指页面具有相同的协议、域名、端口号，有一项不同就不是同源。 有同源策略能够保证 web 网页的安全性。

介绍一下 xss csrf 攻击

XSS 是指跨站脚本攻击。攻击者利用站点的漏洞，在表单提交时，在表单内容中加入一些恶意脚本，当其他正常用户浏览页面，而页面中刚好出现攻击者的恶意脚本时，脚本被执行，从而使得页面遭到破坏，或者用户信息被窃取。

要防范 XSS 攻击，需要在服务器端过滤脚本代码，将一些危险的元素和属性去掉或对元素进行 HTML 实体编码。cookie 加 httponly

CSRF 是跨站请求伪造，是一种挟制用户在当前已登录的 Web 应用上执行非本意的操作的攻击方法

它首先引导用户访问一个危险网站，当用户访问网站后，网站会发送请求到被攻击的站点，这次请求会携带用户的 cookie 发送，因此就利用了用户的身份信息完成攻击

防御 CSRF 攻击有多种手段：

为表单添加校验的 token 校验
cookie 中使用 sameSite 字段
服务器检查 referer 字段

什么是强缓存？什么是弱缓存？

强缓存：直接从缓存读取，不发送请求到服务器。 HTTP 头字段：Expires（绝对时间）、Cache-Control（相对时间）

弱缓存：发送请求到服务器，验证资源是否有更新（304），决定是否使用缓存。HTTP 头字段：Last-Modified / If-Modified-Since（资源的最后修改时间）、ETag / If-None-Match（资源的唯一标识（通常是一个哈希值））

此外，还要注意很重要的一点就是，babel 只是转译新标准引入的语法，比如 ES6 的箭头函数转译成 ES5 的函数；而新标准引入的新的原生对象，部分原生对象新增的原型方法，新增的 API 等（如 Proxy、Set 等），这些 babel 是不会转译的。需要用户自行引入 polyfill 和 runtime 来解决

直接使用 babel-polyfill 对于应用或页面等环境在你控制之中的情况来说，并没有什么问题。但是对于在 library 中使用 polyfill，就变得不可行了。因为 library 是供外部使用的，但外部的环境并不在 library 的可控范围，而 polyfill 是会污染原来的全局环境的（因为新的原生对象、API 这些都直接由 polyfill 引入到全局环境）。这样就很容易会发生冲突，所以这个时候，babel-runtime 就可以派上用场了。Babel Runtime 提供了一种“沙箱”机制，通过避免全局污染来确保代码更加模块化和独立性。

什么是虚拟 DOM， 有什么作用，为什么可以提高性能？

虚拟 DOM 分三步运作

  1. 用 javascript 对象模拟真实的 dom 树，对 dom 进行抽象。
  2. diff 算法——比较两个虚拟 dom 树的差异
  3. patch 算法——将两个虚拟 dom 对象的差异更新到真实的 dom 树。

虚拟 DOM 可以降低浏览器的性能的消耗：javascript 的运行速度远大于 dom 操作的执行速度，因此，运行 patching 算法来计算出真正的需要更新的节点，最大限度地减少了 dom 操作。通过 diff 算法，减少回流和重塑，通过 diff 算法优化遍历，对真实的 dom 进行打补丁式的新增、修改、删除、实现局部更新，减少回流和重绘等操作。用 diff 算法，减少了回流和重绘。

函数组件和类组件的区别？

  1. 函数式组件是函数式编程思想，主打的是 immutable（不可变的）、没有副作用、引用透明等特点；而类组件是面向对象编程思想，它主打的是继承、生命周期等核心概念。官方更支持使用函数式组件。
  2. 状态管理，函数式组件使用 useState，类组件使用 this.setState
  3. 生命周期，函数式组件使用 useEffect 通过不同传参形式来实现，类组件有各种继承而来的钩子去实现。
  4. 调用方式，函数式组件只需要执行函数即可，类组件需要经过实例化的过程。
  5. 获取渲染的值，根本区别：函数式组件捕获了渲染所用的值。函数式组件的 Props 是不可变(immutable)的而类组件的 this 是，而且永远是，可变(mutable)的

简述微信小程序原理

小程序分为两个部分 webview 和 appService。其中 webview 主要用来展现 UI，appService 有来处理业务逻辑、数据及接口调用。它们在两个进程中运行，通过系统层 JSBridge 实现通信，实现 UI 的渲染、事件的处理。

JSBridge 在当中的作用？

JSBridge 充当逻辑层和视图层之间的桥梁，允许这两个线程安全地交换数据和消息。当逻辑层需要更新视图层时，会通过 JSBridge 将数据发送到视图层进行渲染。反之，视图层也可以通过 JSBridge 向逻辑层发送用户交互事件（如点击、输入等）。

JSBridge 的本质是什么？

JSBridge 的本质是在原生端通过对 JavaScript 环境中的 window 对象注入特定的方法来实现通信和调用。

flutter 的路由跳转有几种方式？

Navigator.pushReplacement 用新的路由替换当前路由，通常用于登录成功后跳转到主页面。

Navigator.pushAndRemoveUntil 推送一个新路由，并移除栈中所有的路由，直到满足特定条件。

flutter 的更新机制是怎么样的？

Widget 只是一个配置且无法修改，而 Element 才是真正被使用的对象，并可以修改。
当新的 Widget 到来时将会调用 canUpdate 方法，来确定这个 Element 是否需要更新。
canUpdate 对两个（新老） Widget 的 runtimeType 和 key 进行比较，从而判断出当前的 Element 是否需要更新。若 canUpdate 方法返回 true 说明不需要替换 Element，直接更新 Widget 就可以了。

flutter 的 diff 规则是怎么样的？

检查 Widget 类型和 Key：

如果新的 Widget 与旧的 Widget 的 runtimeType 和 Key 相同，则认为它们是同一个 Widget，可以复用相应的 Element 和 RenderObject。
否则，会创建新的 Element 和 RenderObject 替换旧的。

为了提升性能 Flutter 的比较算法（diff）是有范围的，它并不是对第一个 StatefulWidget 进行比较，而是对某一个层级的 Widget 进行比较。

flutter 的 stream 有什么作用？

Stream 类可以用于监听和响应数据的变化，非常适合需要持续更新或实时数据的场景。

Stream 有几种？

Stream 有两种类型：单订阅 Stream（Single Subscription Stream）和广播 Stream（Broadcast Stream）。

单订阅 Stream（Single Subscription Stream）
特性

  1. 单一订阅者：每个单订阅 Stream 只能有一个监听器（listener）。在一个监听器取消订阅之前，不能有其他监听器订阅同一个 Stream。
  2. 按顺序处理数据：数据按顺序发送给订阅者，并且订阅者会逐个处理这些数据。
  3. 典型用途：适用于一次性处理数据流的场景，比如文件读取、HTTP 请求等。

广播 Stream（Broadcast Stream）
特性

  1. 多个订阅者：广播 Stream 可以有多个监听器。所有监听器会同时接收数据。
  2. 按顺序处理数据：数据会按顺序发送给所有的订阅者。
  3. 典型用途：适用于需要多次订阅的场景，比如事件总线、实时数据更新等。

Widget 更新的总体流程

  1. 调用 setState 或 StatefulWidget 的 build 方法：触发重建。
  2. 构建新的 Widget 树：框架根据新的状态构建一个新的 Widget 树。
  3. Diff 过程：
    比对旧的 Widget 树和新的 Widget 树：找到需要更新的部分。
    更新 Element 树和 RenderObject 树：根据比对结果更新这两个树。

vue3 比 vue2 有什么优势？

  1. 性能更好，Vue 3 使用 ES6 的 Proxy 来实现响应式系统，而不是 Vue 2 中的 Object.defineProperty。这使得 Vue 3 可以更有效地追踪依赖，减少内存占用并提高性能。
  2. 打包体积更小，Vue 3 的构建方式更加现代，支持 Tree Shaking。这意味着在构建时，未使用的代码会被自动剔除，从而减小最终的包体积。
  3. 更好的 ts 支持，Vue 3 对库进行了全面重写，设计之初就考虑了 TypeScript 的支持。这使得 TypeScript 类型推导更加准确，开发体验更好。
  4. 更好的代码组织和逻辑抽离，有 Composition API

谈谈你对 vue 的理解，或者说 vue 最重要的是什么？

第一个就是数据驱动视图，vue 出生前的 jQuery 时代，仍然需要操作 dom 去操作视图。vue 的数据响应式能由数据驱动视图的变更。

第二个是独特的指令系统和声明式渲染，Vue 基于标准 HTML 拓展了一套模板语法，使得我们可以声明式地描述最终输出的 HTML 和 JavaScript 状态之间的关系。v-if,v-for 这些指令在开发中更好地完成数据驱动视图的逻辑。

第三个是组件化，细化开发单元，抽离了重复的开发区域并加以复用，降低整个系统的耦合度，并且借此来对业务进行和代码按职责进行分离，有更好的开发、调试和维护体验。

Proxy 和 Object.defineProperty 的区别？

Proxy 可以代理整个对象，包括对象的所有属性、数组的所有元素以及类似数组对象的所有元素。而 Object.defineProperty 只能代理对象上定义的属性。
Proxy 可以监听到新增属性和删除属性的操作，而 Object.defineProperty 只能监听到已经定义的属性的变化。

为什么在 vue2 时代没有启用 proxy?

因为那个时代的 Proxy 还没有成熟 并且使用 Object.defineProperty 的响应式系统已相对成熟
