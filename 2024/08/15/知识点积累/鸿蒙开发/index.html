鸿蒙开发

1.你怎么理解鸿蒙开发？

就是 flutter 的声明式 UI 和布局，小程序的路由机制，Vue3 的组件化和双向绑定，TS 的类型混在一起

2.在开发中遇到什么是跟前端开发习惯不一样的地方？

绝对定位不会浮到最上层级而是根据书写顺序层叠

3.怎么样通过鸿蒙去实现小程序的功能？

通过 stage 模型的 UIAbility 组件去拉起小程序，因为一个应用可以拥有多个 UIAbility,通过上下文对象 context 的 startAbility 方法就可以拉起小程序，通过 request.downloadFile 下载小程序包，通过 DOM 树映射原生 viewTree,调起加载代码原生 native 引擎做原生控件渲染。

4.应用侧的原生和 WebView 如何进行通讯？

通过 Webview 的 WebviewController 的 API 可以实现，其中 loadUrl 可以动态加载不同的资源（可以把小程序的包拉下来加载成小程序），其中 registerJavaScriptProxy 可以注入 JavaScript 对象到 window 对象中，并在 window 对象中调用该对象的方法。其中 runJavaScript 可以让原生侧调用 JavaScript 脚本中的方法。其中 createWebMessagePorts 可以创建 Web 消息端口，然后可以用 postMessage 和 html 通讯，html 使用 postMessage 发送，onmessage 接收，原生侧使用 postMessage 发送，postMessageEvent 接收

5.关于公众号怎么看？

跟公众号服务器进行通讯实现问答和消息交互，必要的时候打开小程序。

6.如何实现搜索功能？

第一个是 webview 进行网络搜索，第二个是通过文件系统（file.fs 进行文件访问）和网络系统（net.http 进行 HTTP 请求）对，收藏和聊天记录做正则匹配搜索。

7.如何封装 sdk 包？

我印象中就好像封装一个前端工具包一样吧，不过有一些像微信登录这种需要调用 stage 模型去拉起其他应用或者模块这些是原生独有逻辑。

8.askts 和 ts 的关系？

大部分一样，有些不一样，第一个不支持运行的时候更改对象布局，就是说原来通过类型定义好一个对象是什么样，有什么成员，不能中途去给他加减更换。对象里的属性必须标注类型，没有鸭子效应，一个类型 A 有另一个类型 B 的一部分属性，不代表 A 在业务的时候能让代替 B 去工作。

9.arkts 或者 ts 和 js 源码在编译时使用的时什么底层？

ts/js 运行在方舟编译器中，最终编译成字节码，方舟字节码（Ark Bytecode），是由方舟编译器编译 ArkTS/TS/JS 生成的，提供给方舟运行时解释执行的二进制文件，字节码中的主要内容是方舟字节码指令。字节码已直达机器指令。如果需要对 ts 和 JavaScript 做传统的支持，需要应用自带 v8 引擎并根据提供的 native 工具链来运行。

10. 鸿蒙是如何渲染 UI 的？

首先他通过好像 Vue 一样的 JS 架构层让开发者构建应用代码，然后使用 JS 引擎把代码转成字节码，然后调用他的渲染引擎进行渲染。
