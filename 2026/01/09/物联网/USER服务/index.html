报错

原因

  1. 并发问题：在高并发环境下，两个请求可能同时执行到删除旧 token 的步骤，然后同时插入新 token，导致其中一个请求失败。
  2. 事务隔离级别问题：如果事务隔离级别设置不当，可能会导致一个事务看不到另一个事务已经删除的记录。

解决方案

  1. 使用数据库的 UPSERT 操作（INSERT ON DUPLICATE KEY UPDATE），这样可以在一个原子操作中完成更新或插入。
  2. 或者在应用层使用更严格的锁机制，确保同一时间只有一个请求可以处理同一个用户和设备的 token。
