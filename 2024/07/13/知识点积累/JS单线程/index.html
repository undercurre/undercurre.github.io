JS单线程

如何理解 JavaScript 的单线程？

JavaScript 的运行环境（如浏览器、Node.js）采用单线程模型，这意味着在同一时间只能执行一个任务。这个线程通常被称为“主线程”。但是主线程外可以跑异步任务，这些异步任务由独立于主线程的其他线程计算，并将结果通过回调函数传入主线程。

浏览器有哪些主线程以外的线程?

第一个是定时器（Timers）线程：例如 setTimeout 和 setInterval，这些定时器的实现通常由浏览器的定时器线程来处理。当时间到了，定时器线程会将回调函数添加到任务队列中，等待主线程处理。

第二个是网络请求（Network Requests）线程：例如 fetch 和 XMLHttpRequest，网络请求由浏览器的网络线程处理。当网络请求完成时，网络线程会将回调函数添加到任务队列中，等待主线程处理。

事件监听（Event Listeners）线程：用户交互事件（如点击、滚动等）由浏览器的事件线程处理。当事件发生时，事件线程会将回调函数添加到任务队列中，等待主线程处理。

Web Workers：Web Workers 允许你在后台创建独立线程中运行 JavaScript，独立于主线程。你可以使用 Web Workers 来执行计算密集型任务，而不会阻塞主线程。

Nodejs 有哪些主线程以外的线程?

Nodejs 由 libuv 的线程池处理异步操作：例如文件系统操作（如 fs.readFile）和网络操作（如 http.request），这些操作由 libuv 的线程池处理。当操作完成时，线程池会将回调函数添加到任务队列中，等待主线程处理。例如 setTimeout 和 setInterval，这些定时器由 libuv 处理。定时器到期后，libuv 会将回调函数添加到任务队列中，等待主线程处理。Node.js 中的异步任务由 libuv 的事件循环处理。事件循环不断检查任务队列，将回调函数调度到主线程执行。

主线程怎么保证自己不被阻塞？

虽然 JavaScript 是单线程的，但它可以通过事件循环（Event Loop）机制处理异步任务，确保不会阻塞主线程。

如何理解 Nodejs 适合 IO 密集型后端处理？

因为 JS 有事件驱动架构，事件驱动的应用程序能够把 IO 操作独立于主线程以外进行，避免阻塞。
