cookie和session的异同

cookie 存储在客户端浏览器中，通常每个 cookie 的大小限制为 4KB 左右，每个域名最多可以存储 20 个 cookie，不同浏览器可能有所不同，因为存储在客户端，容易被用户查看和篡改，需要采取安全措施，可以设置过期时间，如果不设置过期时间，cookie 将在浏览器会话结束时删除。可以设置 cookie 的作用域，限定哪些 URL 可以访问该 cookie，通常使用于保存用户偏好设置（如语言、主题），保存用户登录状态，跟踪用户行为（如广告投放、访问统计）。

session 存储在服务器端，客户端仅保存一个 Session ID，服务器端存储数据，大小只受服务器存储能力限制，因为存储在服务器端，更安全，但需要保护 Session ID（例如通过 HTTPS 传输），通常在用户会话结束后（如关闭浏览器、会话超时）失效，可以通过配置服务器来管理会话超时时间。Session ID 通过 cookie 或 URL 传递，与特定会话绑定。通常使用于保存用户登录状态和会话数据，在多页面之间传递用户数据（如购物车、表单数据），管理用户权限和访问控制。

相同点
在于状态管理：都用于在无状态的 HTTP 协议上管理用户状态。数据存储：都可以用于存储用户的会话数据。

不同点
在于存储位置：
Cookie 存储在客户端浏览器中。
Session 存储在服务器端，客户端只保存 Session ID。
安全性：
Cookie 相对不安全，易被篡改，需要加密和校验。
Session 更安全，因为数据存储在服务器端，但需要保护 Session ID。
生命周期：
Cookie 可以设置长期有效（如几天、几个月）。
Session 通常是短期的，会话结束后即失效。
数据容量：
Cookie 有大小和数量限制。
Session 存储在服务器端，没有明显限制，只受服务器存储能力限制。
