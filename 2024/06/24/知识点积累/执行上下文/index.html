执行上下文

解释执行上下文的概念，以及它在 GS 当中的重要性。

执行上下文是 JS 当中一个非常核心的概念，它描述了代码在执行时的环境，包括变量、函数的定义、作用、域链等等一些信息，那么嗯，我们就来了解一下它都有哪些执行上下文。好，首先它会有全局执行上下文以及函数执行上下文。全局执行上下文就是当 JS 代码开始执行的时候，首先会创建一个全局执行上下文，它是在浏览器环境当中，那么这个上下文它是和 window 对象关联的，并且它还是程序当中默认的执行环境。任何不在函数内部的代码，它都在这个环境里面去执行，那么在这个上下文当中它会自动创建全局变量对象 window，并且 this 会指向这个全局变量，这个就是全局指向上下文。

那么函数执行上下文就是每个函数被调用的时候，一个新的函数执行上下文就会被创建，那么这也就意味着每次函数调用，即使是对同一个函数的多次调用，都会拥有各自独立的执行上下文。那么在函数上下文当中， this 的值它会根据函数的调用方式动态改变，那么同时还会创建一个新的这个环境来存储函数内部的变量和一些参数。

OK，那接下来我们就要来聊一下，就是上下文的生命周期。好，每个上下文它都有一定的生命周期，包括你创建、执行、销毁。好，那么在创建阶段它会有这样几一些操作，就是创键变量、对象、存储函数、参数、局部变量、函数声明和变量声明等等。紧接着它要确定 this 的值，再往下它要创建作用域。这个创建作用域就是用来解析变量和函数的这个引用，那么它链接着当前执行上下文和外部执行上下文的这个作用域，OK，那么它的重要性，重要性。首先就是作用域和变量解析，因为它决定了变量和函数的可访问性以及 this 的值，那么这对于理解 JS 当中作用域是比较重要的。

第二个是内存管理，我们可以通过执行上下文，这个 JS 引擎能够追踪变量的生命周期，适当的分配和释放内存。那么还有就是函数调用和闭包，执行上下文是实现函数嵌套和闭包的基础，闭包能够访问它父级执行上下文的变量，即便父极函数已经执行完了，这个就是闭包。总的来说，执行上下文给 JS 提供了一个框架，帮助他去解释代码执行的时候该怎么去执行，对吧？那么也是我们去理解 JS 语言机制的一个基础，OK，这个就是关于执行上下文的概念以及它的一个重要性。
