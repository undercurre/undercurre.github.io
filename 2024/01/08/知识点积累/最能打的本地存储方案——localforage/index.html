最能打的本地存储方案——localforage

背景

如果你需要一个存储量大，还要覆盖全多种设备多种浏览器的本地离线存储方案

方案分析

  1. 既然要存储的数量大，得排除 cookie, 因为只有 4KB

  2. localStorage，虽然比 cookie 多，但是同样有上限（5M）左右

  3. websql 使用简单，存储量大，已经被废弃，兼容性差

    WebSQL 是一个在浏览器中提供关系型数据库访问的 API。它是由 Web SQL Database 规范定义的，该规范已经被停止维护，不再是 W3C 的标准。

  4. indexedDB api 多且繁琐，存储量大、高版本浏览器兼容性较好

    websql 的替代方案，IndexedDB 是一种在浏览器中提供客户端数据库存储的 API，它是 W3C 制定的标准。IndexedDB 允许你在浏览器中异步地存储和检索大量数据，并提供了比以前的技术（如 WebSQL 和 Web Storage）更灵活和强大的存储机制。

localforage 应运而生

localForage 是一个 JavaScript 库，只需要通过简单类似 localStorage API 的异步存储来改进你的 Web 应用程序的离线体验。它能存储多种类型的数据，而不仅仅是字符串。

安装

兼容性

localForage 有一个优雅降级策略，若浏览器不支持 IndexedDB 或 WebSQL，则使用 localStorage。在所有主流浏览器中都可用：Chrome，Firefox，IE 和 Safari（包括 Safari Mobile）。兼容性方面 loaclForage 基本上满足 99%需求

存储量

首先 indexDB 的存储，理论上是硬件有多大内存就可以存多少，但是有些浏览器厂商会限制，具体限制各家不同，但是基本最小是 250M 起步

使用

获取存储

getItem(key, successCallback)

从仓库中获取 key 对应的值并将结果提供给回调函数。如果 key 不存在，getItem() 将返回 null。

设置存储

setItem(key, value, successCallback)

将数据保存到离线仓库。你可以存储如下类型的 JavaScript 对象：

  1. Array
  2. ArrayBuffer
  3. Blob
  4. Float32Array
  5. Float64Array
  6. Int8Array
  7. Int16Array
  8. Int32Array
  9. Number
  10. Object
  11. Uint8Array
  12. Uint8ClampedArray
  13. Uint16Array
  14. Uint32Array
  15. String

删除存储

removeItem(key, successCallback)

从离线仓库中删除 key 对应的值。

清空存储

clear(successCallback)

从数据库中删除所有的 key，重置数据库。

localforage.clear() 将会删除离线仓库中的所有值。谨慎使用此方法。

特别问题

内存不足状态下使用

抛出错误 QuotaExceededError 的 DOMError

存储数据出现了脏数据或者读取问题

解决方案

存储数据的时候加上存储的时间戳和模块标识，加时间戳一起存储

遇到内存不足的情况，则根据时间戳和模块标识清理一部分旧数据（内存不足的情况还是比较少的）或者上传数据库,记住提示弹窗。
在用户手机上产生脏数据的情况，想要清理的这种情况的 处理方式是：
  1. 让后端在用户信息接口里面加上缓存有效期时间戳，当该时间戳存在，则前端会进行一次对本地存储扫描
  2. 在有效期时间戳之前的数据，结合模块标识，进行清理，清理完毕后调用后端接口上报清理日志
  3. 模块标识的意义是清理数据的时候，可以按照模块去清理（选填）
  4. 没有后端信息的情况下，根据客诉，使用事先准备好的开发者页面去清除指定模块，报错模块，时间段等
