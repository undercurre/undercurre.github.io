解决被墙的Page们

解决被墙的 Page 们

背景

相信大家都用过 Github Page 来部署过个人博客, 这里有一个痛点：会被墙

解决方案

1. 转到 Gitee

对国内友好了，但是这又有一个很逗比的实名认证

2. 使用 vercel（覆盖国内） + 自定义域名（提速） + cloudflare（起飞）

首先在 vercel 上部署 github 的 github page 下的 blog

然后到 godaddy 找个域名在 vercel 上使用

然后把域名的 DNS 伺服器改到 cloudflare

个中原理

被墙：

中国的网络审查系统，通常被称为“Great Firewall”（GFW），是一个庞大的技术系统，旨在对互联网流量进行监控、过滤和控制。GFW 使用各种技术手段来实施网络审查，包括但不限于：

  1. 域名过滤: GFW 会屏蔽特定网站的域名，使其无法被访问。这通常通过 DNS 污染和 DNS 劫持等方式实现。

  2. IP 地址封锁: GFW 会封锁特定网站的 IP 地址，使其无法在中国访问。这可能会包括 GitHub Pages 所用的 IP 地址范围。

  3. 关键词过滤: GFW 会检测和过滤互联网上的敏感关键词，包括网页内容中的文字、图片、视频等。如果某个网页包含被屏蔽的关键词，访问者可能无法正常访问该网页。

  4. 协议封锁: GFW 可能会封锁特定协议的使用，如 VPN、代理服务器等。这些工具可以用来绕过其他封锁措施，因此中国政府也会对它们进行封锁。

  5. 深度封包检测: GFW 还可以通过检测互联网数据包的内容来识别和阻止特定类型的流量，以实施更精细化的审查和封锁。

综合以上各种技术手段，中国的网络审查系统能够实现对互联网访问的高度控制和过滤，使得许多国外网站和服务在中国大陆地区无法正常访问。这也包括了 GitHub Pages 等一些网站和服务。

VPN：

VPN（Virtual Private Network，虚拟专用网络）是一种通过在公共网络上建立加密隧道，将用户的网络流量路由到远程服务器并加密传输的技术。VPN 可以有效地隐藏用户的真实 IP 地址，并且加密用户的数据流量，提供了一种绕过网络封锁和过滤的方法。以下是 VPN 能够访问被墙资源的一些原因：

  1. 绕过 IP 封锁: VPN 通过将用户的网络流量路由到 VPN 服务器，然后再由 VPN 服务器发送请求到被封锁的资源，使得用户的真实 IP 地址被隐藏，从而绕过了基于 IP 地址的封锁。

  2. 加密通信: VPN 使用加密技术对用户的数据流量进行加密，这使得审查者无法直接读取或识别数据包的内容。即使被墙的资源上有关键词过滤等技术，VPN 加密后的流量也不容易被审查系统检测到。

  3. 更改地理位置: VPN 允许用户连接到遍布全球的服务器，因此用户可以选择连接到其他国家的服务器，使得他们的网络流量看起来是来自于其他国家，而不是被封锁的地区。这样可以绕过地理位置限制，访问被封锁的资源。

  4. 防止深度封包检测: VPN 可以防止审查者对数据包进行深度封包检测，因为数据包经过加密后，审查者无法直接识别数据包的内容，从而难以对 VPN 流量进行过滤和封锁。

需要注意的是，尽管 VPN 可以帮助用户绕过网络封锁，但在某些国家或地区，使用 VPN 可能也会受到限制或违法，因此使用 VPN 时需要遵守当地法律法规。

DNS:

DNS 解析是指将域名（例如：example.com）转换为与之对应的 IP 地址（例如：192.0.2.1）的过程。这个过程是互联网上进行通信的基础之一，它允许用户使用易记的域名来访问互联网上的各种服务和资源，而不需要记住复杂的 IP 地址。

DNS 解析过程大致分为以下几个步骤：

  1. 用户输入域名: 用户在浏览器或其他网络应用程序中输入域名（如 example.com）作为要访问的目标网站的地址。

  2. 本地 DNS 缓存查询: 如果用户的设备（如计算机或手机）中已经存有该域名的 DNS 记录，系统会首先查询本地 DNS 缓存，如果有匹配的记录，则跳过后续步骤，直接使用本地缓存的 IP 地址进行访问。

  3. 本地 DNS 服务器查询: 如果本地 DNS 缓存中没有找到对应的记录，用户的设备会向本地 DNS 服务器发出 DNS 解析请求。本地 DNS 服务器通常由 Internet 服务提供商（ISP）或其他网络运营商提供，它负责将用户的 DNS 请求转发给更高级别的 DNS 服务器。

  4. 递归查询: 本地 DNS 服务器收到用户的 DNS 解析请求后，它会首先查询自己的 DNS 缓存，如果没有找到匹配的记录，它将会向根域名服务器发送一个递归查询请求。根域名服务器是 DNS 解析的起点，它存储了顶级域名（例如.com、.net、.org 等）的 DNS 服务器的地址。

  5. 迭代查询: 根域名服务器收到递归查询请求后，会返回给本地 DNS 服务器一个指向相应顶级域名服务器的 IP 地址的响应。本地 DNS 服务器随后向相应的顶级域名服务器发送一个迭代查询请求，以获取二级域名的 DNS 服务器的 IP 地址。

  6. 获取域名服务器地址: 顶级域名服务器收到迭代查询请求后，会返回给本地 DNS 服务器一个指向二级域名服务器的 IP 地址的响应。本地 DNS 服务器随后向二级域名服务器发送一个查询请求，获取目标域名对应的 IP 地址。

  7. 返回 IP 地址: 最终，本地 DNS 服务器收到目标域名对应的 IP 地址后，会将该地址返回给用户的设备，并且将结果保存到本地 DNS 缓存中，以便下次查询时使用。

  8. 访问目标网站: 用户的设备收到目标域名对应的 IP 地址后，便可以使用该地址向目标网站的服务器发出请求，获取网页内容或其他资源。

这样，通过 DNS 解析，用户可以通过输入易记的域名来访问互联网上的各种服务和资源，而不需要记住复杂的 IP 地址。

二级域名

二级域名是在域名系统中属于主域名下一级的子域名。在一个完整的域名中，通常由多个部分组成，按照从右到左的顺序依次表示不同的层级。例如，在域名”example.com”中，”com”是顶级域名（TLD），”example”是二级域名。

二级域名通常用于对主域名进行更细分或分类。例如，如果”example.com”是一个公司的主要网站，那么可以使用二级域名来表示不同部门或功能。例如：

marketing.example.com

sales.example.com

support.example.com

这些二级域名可以分别指向公司的市场营销、销售和客户支持等不同的部门或功能。二级域名的具体设置和管理通常由域名所有者在域名注册商或 DNS 服务提供商处进行配置。

需要注意的是，二级域名的使用不限于组织内部的功能划分，也可以用于指向不同的网站、应用程序或服务。例如，”blog.example.com”可能是公司的官方博客网站，而”shop.example.com”可能是公司的在线商店。

CDN

CDN（Content Delivery Network，内容分发网络）是一种通过将内容分发到全球各地的服务器，以提供更快速、可靠和安全的网络内容传输服务的技术架构。CDN的核心理念是通过在全球范围内分布内容服务器，将内容缓存到距离用户更近的位置，从而减少用户访问时的延迟和网络拥塞，提高用户的访问体验。

以下是CDN的工作原理和优势：

  1. 内容缓存和分发: CDN在全球范围内部署了大量的服务器节点，这些节点存储了网站的静态资源（如图片、视频、脚本文件等）。当用户请求访问网站时，CDN会将这些静态资源缓存在距离用户更近的节点上，并且将请求路由到最近的节点，从而加速内容的传输和加载速度。

  2. 负载均衡: CDN可以根据用户的地理位置和网络条件，自动选择最佳的服务器节点来响应用户的请求，从而分散网络流量并降低服务器的负载。这可以有效地防止服务器因为突然的流量激增而崩溃。

  3. 提高网站性能: 通过将内容缓存到距离用户更近的位置，CDN可以大大减少内容传输的延迟，加快网站的加载速度，提高页面响应速度和用户体验。

  4. 提高可用性和稳定性: CDN可以在服务器出现故障或网络中断时自动切换到其他可用的节点，从而提高网站的可用性和稳定性。即使某个节点发生故障，用户仍然可以访问其他节点上缓存的内容。

  5. 安全性增强: 一些CDN提供了安全功能，如DDoS攻击防护、SSL加密、内容安全策略等，可以帮助保护网站免受网络攻击和恶意行为的影响。

总之，CDN通过将内容分发到全球各地的节点，并通过负载均衡和内容缓存等技术手段，提高了网络内容传输的效率、可靠性和安全性，为用户提供了更快速、更可靠的访问体验。
