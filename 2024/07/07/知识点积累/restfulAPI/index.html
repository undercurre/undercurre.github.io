RESTFUL API

什么是 restful API？

它是一套 API 设计规范，核心思路是将一切数据视作资源，利用上个视频提到的 HTTP 请求方式描述对资源的具体操作，同时通过 HTTP 响应状态码描述对资源操作的具体结果。期望达到的效果是，看 URL 知道是什么资源，看 method 知道要对资源进行什么操作，看 response code 知道操作是否成功。下面来讲规范的具体内容，关于 method 用于描述对资源的具体操作， get 用于读取资源， pose 用于创建资源， put 用于更新资源， patch 用于局部更新， delete 用于删除资源。关于 URL，因为它用于描述一种资源，因此 URL 中只能使用名词，不能有动词。关于 response 项目中需要保持统一的返回结构，其中 code 需要对齐 http 状态码。下面是我常用的返回结构 code 为状态码， data 为返回数据， message 为状态描述，大家在接口设计的过程中可以适当的做调整，核心是需要对所有接口的返回结构保持统一。

为什么要使用 RESTFUL API？它的好处到底体现在哪里？

首先现在已经不是 ASP JSP 服务器端指出页面的时代，也不是单个服务器只服务于单个前端的时代，因此一套好的接口规范可以大幅降低前后端的沟通成本。其次，后端可以更专注于维护一套通用的接口服务，前端则更专注于业务前后端分离的更加彻底。最后，规范约束了接口具备更强的原子性，把更多的业务灵活回给前端。
