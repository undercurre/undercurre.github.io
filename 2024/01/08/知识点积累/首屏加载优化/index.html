首屏加载优化

首屏加载优化

现象

白屏时间：从打开页面到看到页面，中间白屏停留的时间。

目标

缩短白屏时间，用户能够更快的看到我的页面！

方向

  1. 减少资源体积，从而缩短请求时间

  2. 减少资源请求个数，从而缩短等待时间

减少体积

如何知道打包体积？

打包构建的时候，使用–report 命令：

webpack
  1. 自带命令
  1. Webpack Bundle Analyzer 包
vite
  1. –report
  1. Rollup Plugin Visualizer 包

打包结束后，会在 dist 目录里面生成一个 report html 文件，里面会显示你打包体积分布情况，可以根据项目情况，侧重优化。

删除没有使用到的代码

webpack 使用 webpack-unused-plugin
vite 使用 vite build --report 或者 npx esbuild --tree

Sourcemap（源映射）是一种文件，它保存了原始源代码与转换后的代码之间的映射关系，用于在调试时将压缩、合并或转换后的代码映射回原始的、易读的源代码。

删除 prefetch(极其需要的情况下才使用，预加载本身就用于提高体验)

prefetch 是一种浏览器预加载资源的机制，用于提前加载可能在将来需要的资源，以加速页面的加载和导航。通过 prefetch，浏览器可以在空闲时预先下载一些资源，提高用户体验。

vite
webpack

如果是直接使用 import() 语法，Webpack 默认会根据配置进行处理。但你也可以通过使用 /** webpackPrefetch: true */ 或 /** webpackPrefetch: false */ 注释来手动控制是否预加载。
