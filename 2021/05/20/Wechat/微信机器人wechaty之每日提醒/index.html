需求定义

每日定时 8：00 发送天气及 git-contributions 信息到个人微信

方案设计

  1. 定时发送业务实现方式使用 node-schedule
  2. 业务可以使用爬虫爬取页面数据或者做接口转发实现方式使用 superagent
  3. 避免机器人检测——每次回复做不定时延时
  4. 重点针对如何在 wechaty 的 API 中获取并过滤出用户或者群，避免微信其他机器人把 AI 的 token 全部洗掉
  5. 收集数据，指定剧本，放给 AI 来做智能建议
