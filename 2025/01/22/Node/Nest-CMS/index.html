Nest-CMS

CMS System development Record

技术选型标准：

  1. Mysql 8.0.40 关系型数据库
  2. Nestjs： 基于 express.js 的 node 后端服务框架
  3. Geeker Admin：Vue3 的后台管理框架
  4. Vue3：渐进式前端框架，用于制作 app 用的 h5，实现移动端功能

数据模型

定义一个名为 mova_cms 的数据库

  1. product 表
  • id: int 类型，主键，自动递增。
  • model: varchar(255) 类型，产品型号。
  • imageUrl: varchar(255) 类型，产品图片 URL，允许为 NULL。
  • sellingPoints: varchar(255) 类型，产品卖点。
  • manualOssUrl: varchar(255) 类型，产品说明书的 URL，允许为 NULL。
  • name: varchar(255) 类型，产品名称，使用 utf8mb4 字符集。
    索引：
  • 主键 id。
  1. consumable 表
  • id: int 类型，主键，自动递增。
  • model: varchar(255) 类型，耗材型号。
  • name: varchar(255) 类型，耗材名称。
  • description: text 类型，耗材描述，允许为 NULL。
  • productId: int 类型，外键，引用 product 表的 id，允许为 NULL。
  • imageUrl: varchar(255) 类型，耗材图片 URL，允许为 NULL。
  • productLink: varchar(255) 类型，耗材的产品链接，允许为 NULL。
    索引与约束：
  • 外键 FK_46308fa8a011371ceb6e6588a1b 关联到 product 表的 id 字段。
  1. user 表
  • id: int 类型，主键，自动递增。
  • username: varchar(255) 类型，用户名，唯一。
  • password: varchar(255) 类型，用户密码。
  • thirdPartyId: varchar(255) 类型，第三方用户标识，允许为 NULL。
  • isActive: tinyint 类型，用户是否激活，默认为 1。
  • email: varchar(255) 类型，用户邮箱，唯一，允许为 NULL。
  • role: enum(‘admin’, ‘user’) 类型，用户角色，默认为 user。
  • createdAt: datetime(6) 类型，记录创建时间，默认当前时间。
  • updatedAt: datetime(6) 类型，记录更新时间，默认当前时间，并在更新时自动变更。
    索引：
  • 唯一索引 IDX_78a916df40e02a9deb1c4b75ed 针对 username 字段。
  • 唯一索引 IDX_e12875dfb3b1d92d7d7c5377e2 针对 email 字段。
    移动端
  1. 移动端就是正常的页面构造 pnpm create vue
  2. 然后引入了一个原子化 css 来加速页面的构建速度 unocss
  3. 新建一个文件夹 api 来定义接口和接口类型, 用 axios 封装一个简单的 request 发起请求
  4. 路由加了固定前缀 markH5,通过动态路由来访问页面，路由参数是产品 id
  5. vite 部署 proxy 开发代理
  6. 打包路径根据开发和生产区分使用相对和绝对路径

后台管理

  1. Geeker Admin 使用起来就简单多了，很多东西都是封装好的
  2. 在文件夹 api 来定义接口和接口类型
  3. 在 assets 换一下需要替换的图片资源
  4. 在 views 添加新页面或者改改旧页面
  5. 因为 Geeker Admin 的路由是动态请求实现的，开发阶段，该请求可以使用 assets/json/authMenuList.json 代替，在这个地方伪造路由参数请求结果。
  6. 通过.env.development 的 VITE_PROXY = [[“/api”,”开发过程中的本地后端端口”]]配置开发代理
  7. 修改环境参数 production 环境下的 VITE_PUBLIC_PATH = ./,VITE_API_URL = “你要请求的地址”

后端

  1. 使用的是 Nestjs
  2. 根据数据结构，编写 dto，entity，controller，service，module
  3. 使用 interceptor 对响应进行规范化
  4. 使用 middleware 打印请求日志
  5. 注意：每个 module 都需要 import 数据库 orm 实例如【TypeOrmModule.forFeature([Product])】
  6. 在 app.module.ts 需要集中配置数据库，拦截器，中间件
  7. 在 main.ts 启用 CORS 解决跨域问题，加入 Swagger，改端口
  8. build 加– wepack 可以打包成一个文件，但注意：这里不打包依赖

Mysql: 使用 8.0.40

Nginx: 使用 1.27.0
