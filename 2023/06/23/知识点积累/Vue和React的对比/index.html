Vue和React的对比

Vue 和 React 的对比

相同点

  1. 都有组件化思想
  2. 都支持服务器端渲染(vue-server-renderer、react-dom/server)
  3. 都有 Virtual DOM（虚拟 dom）
  4. 数据驱动视图
  5. 都有支持 native 的方案：Vue 的 weex、React 的 React native
  6. 都有自己的构建工具：Vue 的 vue-cli、React 的 Create React App

区别

  1. 组件化通信的不同。react 中我们通过使用回调函数来进行通信的，而 Vue 中子组件向父组件传递消息有两种方式：事件和回调函数

  1. diff 算法不同。react 主要使用 diff 队列保存需要更新哪些 DOM，得到 patch 树，再统一操作批量更新 DOM。Vue 使用双向指针，边对比，边更新 DOM。

    总体来说，vue 的算法更高效，并有效得益于双端到中算法，react 的 Fiber 暂时没有反向链表支撑同样的做法，也没有意向朝这个方向改进

    patch 函数都是用于比对新旧虚拟 DOM 树的变化，并将这些变化应用到实际的 DOM 上。

    这里的字母表示虚拟 DOM 节点，相同字母表示相同类型的节点。现在我们来看看单端比较和双端比较的情况。

    单端比较
    假设我们只使用单端比较，从头部开始比较。首先，我们比较旧头和新头（A 和 A），它们相同，我们继续向后比较。接着，我们比较旧节点 B 和新节点 X，发现它们不同。于是我们会继续向后比较，直到找到匹配或者遍历完整个数组。

    在这个例子中，我们需要比较所有节点才能找到不同之处，因为新节点 X 在旧节点 B 之后。这导致了不必要的比较，因为在 B 之后的节点都不可能匹配。

    双端比较
    现在，我们来看看双端比较。我们同时从头部和尾部开始比较。首先，我们比较旧头和新头（A 和 A），它们相同，我们继续向后比较。接着，我们比较旧尾和新尾（E 和 Z），它们不同。

    在这一步，我们已经知道了两端有不同，而不需要遍历中间的节点。因为如果中间有匹配，那么在单端比较时，早就已经发现了。

    因此，在双端比较中，我们能够更早地发现不同，减少了比较的次数。

    这个简单的例子展示了双端比较是如何通过同时从两端开始比较，以尽早发现不同，从而减少比较次数的。在实际的虚拟 DOM diff 算法中，这种优化对于提高性能是很有帮助的。
