css预处理器分析

分析css预处理起的不同：

sass/scss(with dart-sass) ,sass/scss(with node-sass),Less,Stylus

Sass/SCSS (with Dart Sass):

Sass（Syntactically Awesome Style Sheets）是一种CSS预处理器，它提供了比普通CSS更强大和灵活的功能。
Dart Sass是Sass的官方实现，由Dart编程语言开发。它是Dart生态系统中的一个独立项目。

Sass使用缩进的语法，而SCSS（Sassy CSS）是Sass的新语法，使用大括号和分号，类似于普通的CSS。SCSS更接近普通CSS的语法，因此更易于学习和迁移。
Sass/SCSS支持变量、嵌套规则、混入（Mixin）、继承、函数等功能，这使得CSS代码更模块化和易于维护。
Sass/SCSS (with Node Sass):

Node Sass是另一种Sass的实现，它是使用Node.js构建的，并且速度较快。

从功能角度来说，Node Sass与Dart Sass相似，支持Sass/SCSS的全部功能。
Node Sass的安装和使用方式与Dart Sass有些不同，因为它是使用Node.js的包管理工具npm进行安装。
Less:

Less是一种CSS预处理器，与Sass和SCSS类似，提供了类似编程的功能来增强CSS的能力。

Less使用JavaScript进行解析，并将其编译为普通的CSS。因此，在浏览器端使用Less时，需要引入Less的JavaScript文件，而Sass和SCSS在浏览器端不需要。
Less支持变量、嵌套规则、混入（Mixin）等功能，与Sass/SCSS在功能上有些相似，但语法稍有不同。
Stylus:

Stylus是另一种CSS预处理器，具有自己的独特语法。

Stylus的语法非常简洁，使用缩进和无括号的方式，使得代码更为简洁，可读性更好。
Stylus同样支持变量、嵌套规则、混入（Mixin）等功能，虽然语法上与Sass/SCSS和Less有所不同，但功能上是类似的。

决定用Node Sass

  1. 因为大量项目支持，生态好
  2. 能使用npm安装
